## Висновки:
- Я навчився використовувати Pipenv та Poetry для створення ізольованих середовищ, що дозволяє уникати конфліктів між бібліотеками різних проєктів та забезпечує їхню стабільну роботу.
- На практиці опанував роботу з лінтером flake8, що дало змогу привести код до стандарту PEP 8, зробивши його більш чистим та читабельним для інших розробників.
- Я розібрався, як використовувати файл .env для зберігання секретних даних (змінних середовища), що є критично важливим для безпеки реальних вебзастосунків.
- Я зрозумів різницю між основними та розробницькими (dev) залежностями, навчившись групувати їх у Poetry для оптимізації розміру та безпеки фінального продукту.
